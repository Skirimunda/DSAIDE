What to do

Task 1:

  • Set the simulation with a population size of 1000, gamma=50, beta=0.1.
  • Use the fraction of susceptible hosts left at the end of the outbreak and the formula given earlier in this module to compute the reproductive number.

Task 2:

Compare the value for the reproductive number from the simulation with the value you get by directly using the value of the model parameters and the equation for R0.

Task 3:

  • Double the value of beta. Before you run the simulation, what do you expect to see and what do you expect to get for R0?
  • Run the simulation and compute \(R_0\) using the final outbreak size to test your expectations.

Task 3:

  • Now, double the rate of recovery. Think about your expectations for \(R_0\) and the dynamics.
  • Run the simulation to check your expectations. Use the final outbreak size to compute \(R_0\).

Task 4:

Another way to estimate R0 is to determine the rate of increase in infected hosts at the beginning of the outbreak. During the initial phase, new infections increase exponentially according to \(I(t)=I_0 e^{r*t}\). Usually, for any real outbreak, you do not know I0. You can still estimate r by obtaining two values of I(t) at two time points during that initial growth rate and use it to compute the exponential growth rate r. Figure out how to do that. For this model, the growth rate and R0 are related through \(R_0 = 1+rD\), where D is the infectious duration. Use this to determine \(R_0\). You should get essentially the same answer as above.

Task 5:

What is the value of R at the time the outbreak peaks? (It’s only called R0 at the beginning for a fully susceptible population, when we talk about it more generally, I’ll call it just R). Explain how you can find that value for R, both using intuitive reasoning and using the equation for R0 given above (R0 = 1+r*D). Note that at this R value, the outbreak wanes, but people still get infected. What R value would you need to completely halt any further infections?

Task 6:

What would happen if a new ID came along that had an R0 value that was the same as the one you just computed in the previous question (the one at the peak of an outbreak)? Test this with the simulation. Choose parameter values for which you get the right R0 and report what you observe.

Task 7:

So far, we looked at an individual outbreak. Now we introduce births and deaths. For our model, we use a constant birth rate \(\lambda\), and a fixed per-capita death rate \(\mu\). It is possible to have population dynamics (births, deaths) in the absence of an ID. It is useful to first figure out what’s going on there before you introduce the disease dynamics. Choose values for the birthrate and lifespan, run the model and see what the population does in the absence of an infectious disease. Can you figure out the equation that relates births, deaths and steady state level for the population? Specifically, if you have \(\lambda\) births per year, and a person lives on average Y years, can you find an equation that predicts how many people you will eventually have in your population if you run the model long enough? Test it with the model (or use the model to try and figure out the equation). You will also have to increase the simulation time (and maybe increase the time at which results are returned to speed up things) to see the results.

Task 8:

If births (and deaths) are present, you will be able to get multiple outbreaks and endemic states. Set the parameter values such that your hosts have an average life-span of 50 years, set birth rate such that the population is steady at 1000 in the absence of the ID. Everything else should be as in #1. Run the simulation long enough to reach a steady state, and record the number of susceptibles/infectious/recovered. Derive analytic expressions for the steady state values for S, I, R. Compare the values from the simulation with those predicted from the steady state equations.

Task 9:

Use the fraction of susceptibles remaining at steady state to compute R0 as described in the lecture notes. Double the value of gamma, run the simulation again and again compute R0 using the steady state result. Next double the values for both births and death rates and repeat.

Task 10:

R0 quantifies the level of transmissibility of an ID, which determines how many people will become infected or what level of intervention is needed to stop/prevent an outbreak. However, it is important to be aware that R0 says nothing about the timing/dynamics of the outbreak. Choose some parameter values that produce an outbreak (turn off natural births/deaths for that). Then set the infectious duration 4 times as large (rate reduced by factor of 4) and adjust infectiousness-level (gamma) to get the same R0 as before. Run again and compare the results.

Task 11:

Usually one gets R0 from the data/literature and uses it to pick values for the transmission parameter beta. Let’s try that. Go online and find values for the duration of the infectious period and for R0 for SARS, use those values in your simulation, and use them to also compute beta. You can leave the population size at 1000. Run an outbreak, see if things match.

Task 12:

Knowing R is important for control strategies, e.g. for vaccine campaigns.You learned at what value of R an outbreak switches from growth to decline (often called the threshold value) and that for this R value, outbreaks don’t take off. Let’s say you have an ID that enters a new population where everyone is susceptible. That ID has an R0/R=4. Would you expect to see an outbreak? Why? Now let’s assume that we protected half the population through a vaccine. What is the new value for R, i.e. how many people are being infected on average by an infected person after we vaccinated? Is that new value of R low enough to prevent the outbreak? How low does R need to be to prevent an outbreak? What fraction of the population would you need to protect/vaccinate to achieve such an R?

Task 13:

Let’s test the vaccination idea with the computer simulation. Edit the code such that you can allow for a fraction of the susceptibles to be vaccinated before you run the. You could write a loop over different vaccination levels, or just set the vaccination fraction f by hand and run once. After you have done that, set parameters to get an R0=4, run the simulation for 0% vaccination coverage to confirm things happen as you expect (check if the final size of the susceptible/infected population is what it should be). Then repeat for 50% vaccination coverage (check what R you get and if it’s what you expect) and for the level you determined above to be enough to prevent an outbreak. Also check what R is at that level.

Task 14:

If births (and deaths) are present, you will be able to get multiple outbreaks and endemic states. At the endemic state, R=1. If you introduce some control at steady state that reduces R, e.g. by introducing a treatment that increases average rate of recovery, what do you expect happens to the disease? Will it go extinct? Try it in R. Figure out S/I/R at steady state. Then run the simulation with these values - in the absence of any intervention, the model should remain at steady state. Now introduce control by increasing rate of recovery, see what happens to the infected fraction.
